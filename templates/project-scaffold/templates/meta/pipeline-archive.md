## Relationship to `pipeline.md`

`pipeline.md` is the **active** pipeline — in-progress work, upcoming phases, and open technical debt. This file (`pipeline-archive.md`) holds:

- **Completed phases (✅)** — phases where all items are done and validated
- **Resolved technical debt** — debt items fixed and closed

**Rules:**

- Only the Architect moves content from `pipeline.md` to this archive.
- A phase is archivable when **every item** in it is ✅. Phases with any non-✅ item stay in `pipeline.md`.
- Resolved debt items (formerly 🔧, now ✅) are archivable.
- Archived content retains its original formatting and status markers — nothing is edited during the move, only relocated.
- If an archived phase needs rework or a resolved debt item reopens → move it **back** to `pipeline.md` with a note explaining the reopening.
